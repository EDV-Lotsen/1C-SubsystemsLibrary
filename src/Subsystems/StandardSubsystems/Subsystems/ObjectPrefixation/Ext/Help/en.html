The application supports assigning prefixes to the infobase objects. Prefixes are assigned to items of all catalogs and documents that get its codes automatically. Object prefixation depends on company and infobase. The infobase prefix is assigned to all objects, the company prefix is assigned to infobase objects that associate with the company.

In general, a number or code has the following structure: 
"CC" + "BB" + "-" + number/code,
where:

  • "CC" — company prefix (2 characters). It is "00", if the company prefix is not specified.
  • "BB" — infobase prefix (2 characters).  It is "00", if the infobase prefix is not specified.
  • "-" — hyphen, separates the prefix and number (code).


The number (code) with an infobase and company prefix is set when recording an object, only if the code or number is not set manually or was not cleared before recording the object. When print forms are generated, document numbers are output without the company prefix, because the company is usually output to the print form too. Meanwhile, the output number contains the infobase prefix. Leading zeroes are skipped in the output number too.
