This form is used to modify settings of additional attributes (data).

It is accessible via  Add or  Create commands in the Additional attributes (data) list.

Fill in the required fields:

  • Enter brief property Description to be displayed in the applied object windows.
  • To make the additional attribute mandatory during catalog or document preparation, select the Make mandatory check box. For additional data, this check box is not available.
  • Fill in the fields on Main tab:
    •  Select from list using the Value type check box of additional attribute (data);
    • To specify format of additional attributes of Number, Date, Boolean types, go to Format string wizard by clicking the Default format hyperlink (not available for additional data). 
      • To modify the default settings, set value of the Language (Country) field.
      • Select parameters to specify the required format.
      • Any results of these parameter changes will be immediately displayed in visual form in the Preview line.
      • To confirm the parameter changes, click OK.
      • Once the user specifies the desired format, the hyperlink text will change to User-specified format.
      • For more information on formats, see below.
    • In the Tooltip field, type in any explanations or advice on how a specific additional attribute or data field should be filled. Text stored in this field will be displayed as a popup tip when hovering mouse pointer over corresponding additional attribute (data) field.
  • In the Comment field, type in the description to be displayed in the bottom part of selection list used to include additional attributes (data items) into sets. To fill in this field, go to the Comment tab.

Adding values by sample

You can reuse any additional attributes and data from other sets, or add attributes with common value list By sample for multiple attributes of the same type. This allows to save time and efforts when preparing value lists for multiple attributes of the same type with a single action. A list of all available additional attributes and data will be displayed. Use the Select command to assign an additional attribute (data) from this list to an object.

  • Type and all values of the additional attribute (data) set as sample will be copied to the new item. Fill in the required fields.
  • By default, the selected attribute will become common with the sample set. Entering new values will not be allowed in this case. You will be notified that this value list is common with another attribute. Once the additional attribute (data) is saved, you can view the sample attribute via hyperlink and add values to it, if necessary.
  • The attribute created by sample and thus having a common value list can be converted to independent status.
    • Click Modify to display the Modify additional attribute (data) settings window; in this window, use the Attribute with personal value list radio button to change the attribute status and save the changes. Any corresponding references in documents will be automatically updated. After that, entering new values will be allowed for the attribute.
    • Attention: Be careful when converting the attributes already referenced in any documents. Attribute conversion is irreversible. The document update procedure can be performed automatically, but this is time-consuming and error-prone. We recommend that you modify the existing documents by manually deleting the old reference values and adding the new ones instead.

Adding common additional attributes (data)

  • An additional attribute (data) is considered "common" when it is included in multiple sets. This option is used when it is necessary to share both an attribute and its value list. In lists and reports of catalog/document items, the common attributes are grouped together and displayed as a single aggregated attribute in a separate column or field. To add a common attribute, use the Add - Common command. To add a common attribute to multiple sets, select all required sets before using this command.
  • In the available list, you can select an existing common additional attribute (common additional data) or Create a new one. When entering data, follow the standard procedure. Fill in the required fields.
  • Window title bar specifies (Common additional attribute) or (Common additional data).
  • When a common additional attribute (common additional data) is included in one or more sets, the Common attribute (common data) is included in sets hyperlink will be displayed at the bottom of its window.
  • To convert a common attribute to an independent attribute with separate value list, click the Modify button located next to the hyperlink.
    • The Modify additional attribute (data) settings window will be displayed; in this window, use the Attribute with personal value list radio button to change the attribute status and save the changes. Any corresponding references in documents will be automatically updated.
    • Attention! Reconfiguration of additional attributes (data) is irreversible and may take an excessive amount of time depending on the number of documents referencing the attribute. We recommend that you create a new attribute with a specific value list and perform a transition between these attributes manually (by clearing old attribute values and filling in new values).

Additional attribute (data) value types

Additional attribute (data) value types may vary to best fit the usage procedure. The following additional attribute (data) types are available:

  • String - Define Length of string, select Fixed or Variable parameter (if the entered string is shorter than maximum allowed length, the number of characters used to store the string will be reduced automatically). Maximum string length is 1024 characters in this case. If very long strings are expected, select the Unlimited check box. 
    • The input field for additional attributes of String type can be multiline. To use the multiline input field, select the Multiline input field check box and specify number of lines.
  • Number - select Length and Precision of the number. The default number length is 15, and default precision is 3. These values may vary depending on the input data. Keep in mind that number Length includes both integer and fractional parts, as well as the decimal point. Select the Not negative check box if necessary.
  • When entering additional attributes of Number type, you can specify the number format more precisely. 
    • The entered numbers wiil be rounded to conform to values of Length and Precision fields in Format string wizard. 
    • You can select Shift. 
    • You can modify Decimal separator (comma by default), specify Group separator (space by default), and define Grouping.
    • You can select Representation of zeroes, for example, to remind the users that they should fill in the field.
    • You can also select Representation of negative numbers.
    • You can select the Display leading zeroes check box, if necessary. Then, if the actual number length is less than the maximum length, the remaining leading digits will be filled in with zeroes. 
  • Date - select date content: Date, Time, or Date and time.
    • To specify date format for additional attributes, click the Default format hyperlink. The available formats are mm/dd/yyyy, yyyy-MM-dd, or Local date format (such as "May 22, 2013"). You can also select the Representation of empty dates check box to enable text warnings (such as "Date not specified") in the empty date fields.  
  • Composite data type - this data type allows to delay the final selection of data type until just before entering data into a catalog. Select the Composite data type check box, then select check boxes for all data types allowed for the attribute.
    • For example, user may be unaware of a specific date, such as the date of company registration. In this case, they should be allowed to choose between Date and String data types. If a precise date is known to the user, they should select Date type and enter the date; otherwise, they should select String type and provide a text description, such as "Early 1900s".
    • Each data type included in the composite data type can be customized as usual, including date format, string parameters, and more.
    • When entering an additional attribute value of composite data type, you have to specify the data type first.
  • Boolean - generally, values like True and False, Yes and No, are used.
    • You can modify the format of these values in Format string wizard. You can specify Representation of boolean False or Representation of boolean True. In this case, the users will be prompted to choose between values specified by you, instead of generic Yes and No.
  • Additional value - this data type is commonly used, prompting users to select a value from the list. This is the default data type. Both standard and grouped value lists are used. Additional attributes (data) of Additional value type can be used to describe any properties of applied objects.
  • Additional value (hierarchy) - this list structure is also common; it is known as "item hierarchy". Items of this list have identical properties but they can have subordinate items. The hierarchy structure can become very complex.
  • User - Values from the Users list can also be used as additional attributes (data).

      Notes on entering additional data of Additional value or Additional value (hierarchy) types

      • Basides the standard fields, the following fields are available:
        • Use weight coefficients for values - select the corresponding check box to enable this option. This field can be useful for analytical and reporting purposes. You are free to decide how you prefer to use weight coefficients in your analytics and reports. This field is optional.
        • Value form title - enter the title for item input windows (hierarchical item input windows). 
        • Value selection form title - enter the title for value selection windows (hierarchical value selection windows). We recommend that you specify personalized titles for additional attribute (data) input and value selection windows, thus facilitating user input.
      • After selecting data type and filling in the Name field, click Save. Then, value list will become available. Click Create to enter values. Fill in the required fields. 
      • If you did not save the entered data before clicking Create, a warning will be displayed. To proceed with entering values, click Save.

      Changing format of additional attributes and data

      After saving, no further changes can be made to value types or format of additional attributes and data. Instead, you will have to delete the existing additional attributes or data, and then create them again, with proper value types.
