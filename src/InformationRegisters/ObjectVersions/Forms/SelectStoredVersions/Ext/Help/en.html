This list is used to view object change history and choose between stored versions. Version history provides quick and clear answers to questions like "Who changed this object", "When was this object changed last", or "What was changed exactly".

This is a useful tool for document or list change history management, object version viewing, and object version comparison.

Before you can start working with object change history, you need to perform the initial object versioning setup. Only the application adimistrator can view or set up version history.

To view version history for a list or a document, execute the Change history command located on its navigation panel. This command is available only to the application administrator, and only for the objects that support versioning.

The list includes:

  • # - version number. Versions are listed in descending order, with version 1 listed last;
  • Change author - name of user that made changes to the object;
  • Date - object change date and time;
  • Comment - detailed information on the object version. This field is filled in automatically. For example, after a version upgrade, new verson number and upgrade date are added to Comment.

Viewing versions

  • Click Open version to view the report describing object status on a given date.

Comparing versions

  • Select two or more versions in the object version list and click Compare versions to view the report describing object changes between the selected versions.

Version rollback

To roll an object back to an earlier version, select a version and click Go to version. Any changes introduced after this version was saved will be canceled. The Comment field is filled in automatically. For example, after a version rollback, new verson number and rollback date are added to Comment.
