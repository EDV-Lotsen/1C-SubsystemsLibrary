This form is used to view and replace data conversion rules. Data conversion rules are used during data synchronization between two applications. 

The default data synchronization settings are applied in accordance with Standard exchange rules. The exchange rules include object conversion rules and object recording rules.

You can access Object conversion rules from the data synchronization settings, via command All actions - Import object conversion rules.

Rule source

Normally, Standard application rules are used for data synchronization between standard applications (name of a conversion rules template used as a source for the standard rules must be specified).  However, in specific cases it may be necessary to use Rules from file (for example, when synchronizing data between infobases with non-standard configuration) or Debug mode (for example, when errors occur during data synchronization).

The Rule source radio button is used to select a source for object recording rules:

  • Standard application rules (used by default).
  • Rules from file. Ensure that the rules you are about to import are created in the current application version. Using rules created in previous versions can cause errors.
  • Debug mode. When you select Debug mode the following options will be available:
    • Attach external data processor for export debug - if you select this option, you wil need to specify path and file name for external data processor containing an export handler debug module created in Data conversion application. In client/server mode, server path is specified.
    • Attach external data processor for import debug - if you select this option, you wil need to specify path and file name for external data processor containing an import handler debug module created in Data conversion application. In client/server mode, server path is specified.
    • Data synchronization logging mode - if this option is enabled, all events and errors that occur during data exchange will be recorded to a log file you select. If you do not select a log file, all messages will be recorded to Event log.
    • Do not interrupt data exchange on errors - if this option is enabled, data exchange procedure will continue uninterrupted upon any errors. Data exchange will complete, and any debug messages will be saved to the exchange log file or event log.

The list of currently active rules is displayed below the radio button.

Rules information

The following rules information is displayed for additional verification:

  • Reminder of errors that could occur due to using rules from external files;
  • Application conversion rules created on - rules creation date and time;
  • Source application - object conversion rules version;
  • Target application - object conversion rules version;
  • Correspondent conversion rules created on - rules creation date and time;
  • Source application - object conversion rules version;
  • Target application - object conversion rules version.

Advanced

  • Select the Use selective object change record filter check box.

Importing rules

  • Set the Rule source radio button.
  • Click Import rules.
  • Specify path to the rules file if necessary.

Exporting rules

You can export the recording rules to a separate file.

  • Click Export rules.
  • Select export method:
    • Normal file;
    • ZIP archive.
  • Specify file path on a local computer or network disk. File name will be generated automatically. Click Save.

Using external rule files instead of standard object conversion rules

  • Set the Rule source radio button to Rules from file or Debug mode. Click Import rules. Select a rule file located on a local computer or network disk.
  • Make sure that rules contained in the selected file are compatible with current versions of source and target applications. Take note of rules creation date displayed in the Rules information field.
  • Click Save and close. The imported rules will apply immediately.

Note: the same object conversion rules and object recording rules are used for all data synchronization procedures of the same type, i.e. based on the same exchange plan (for example, data synchronization procedures Synchronization with Krasnodar branch and Synchronization with Kiev office are of the same type, because they both are based on Full exchange plan). Therefore, to apply new rules to multiple data synchronization procedures of the same type, you only need to import new rules for any one of them; the new rules will automatically apply to all other data synchronization procedures of the same type.
