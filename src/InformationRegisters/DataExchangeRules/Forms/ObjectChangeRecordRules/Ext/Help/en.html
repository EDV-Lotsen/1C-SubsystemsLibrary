This form is used to view and replace data recording rules. Data recording rules are required for all data synchronization procedures.

The default data synchronization settings are applied in accordance with Standard exchange rules. The exchange rules include object conversion rules and object recording rules.

You can access object recording rules from the data synchronization settings, via command All actions - Import object recording rules.

Rule source

The Rule source radio button is used to select a source for object recording rules:

  • Standard application rules (used by default).
  • Rules from file (for debug purposes only).

The list of currently active rules is displayed below the radio button.

Rules information

The following rules information is displayed for additional verification:

  • Full name of rules;
  • Creation date - rules creation date and time;
  • Exchange plan - data synchronization type;
  • Application - full version of the application.

Importing rules

  • Set the Rule source radio button.
  • Click Import rules.
  • Specify path to the rules file if necessary.

Exporting rules

You can export the recording rules to a separate file.

  • Click Export rules.
  • Select export method:
    • Normal file;
    • ZIP archive.
  • Specify file path on a local computer or network disk. File name will be generated automatically. Click Save.

Using external rule files instead of standard object recording rules

  • Set the Rule source radio button to Rules from file. Click Import rules. Select a rule file located on a local computer or network disk.
  • Make sure that rules contained in the selected file are compatible with current versions of source and target applications. Take note of rules creation date and other useful information displayed in the Rules information field.
  • Click Save and close. The imported rules will apply immediately.

Note: the same object recording rules are used for all data synchronization procedures of the same type, i.e. based on the same exchange plan (for example, data synchronization procedures Synchronization with Krasnodar branch and Synchronization with Kiev office are of the same type, because they are both based on Full exchange plan). Therefore, to apply new rules to multiple data synchronization procedures of the same type, you only need to import new rules for any one of them; the new rules will automatically apply to all other data synchronization procedures of the same type.
