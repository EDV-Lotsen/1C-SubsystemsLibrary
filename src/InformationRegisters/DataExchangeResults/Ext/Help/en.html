This form is used to review statistics based on data synchronization results.

Data synchronization results are available as a hyperlink in the Data synchronization list, which can be accessed from the application desktop or other locations. In case of successful data synchronization, the hyperlink displays No warnings; otherwise, it displays Warnings (followed by the number of warnings). To open the detailed warning list, click the hyperlink.

During data synchronization, problems may occur and manual intervention may be required from user. The list contains description of data that caused synchronization problems and the causes of there problems. The causes are briefly described in a separate column; click any cause to display detailed information at the bottom of the list, including the names of list fields or document fields where problems occurred.

Filters

For your convenience, the following filters can be enabled:

  • In the Period field,  Select a specific data period;
  • In the Sender field,  Select a specific infobase from the list;
  • In the Cause field, type in a specific error cause (this filter is not available on the Conflicts tab).

All data synchronization problems can be roughly broken down into the following categories (each category being displayed on a separate list tab):

Unposted documents

This list displays the documents that could not be posted for any reason. To fix a document posting problem, we recommend that you open the document and eliminate any inconsistencies that prevent posting (for example, insufficient product balance). The cause that prevents document posting is displayed at the bottom of the list.

  • To start fixing errors in the selected documents, click Change selected. 
  • Once all errors are fixed, Post the document again.
  • To open the document, click All actions - Change.
  • To hide the document from the list without fixing the posting problems, click Ignore.
  • To display the ignored documents, click All actions - Show ignored.
  • To revert the ignored document, click All actions - Do not ignore.

Blank attributes

This list displays blank attributes of catalog items. To fix this problem, open the related catalog item by double-clicking it and fill in all required fields, then save the item.

  • To fill in the attributes of selected items, click Change selected.
  • To open an item, click All actions - Change.
  • To hide the catalog item from the list without filling in the mandatory attributes, click Ignore.
  • To display the ignored catalog items, click All actions - Show ignored.
  • To revert the ignored catalog item, click All actions - Do not ignore.

Conflicts

When data synchronization between two applications is enabled, the same data may change in both applications nearly simultaneously (between two data synchronization sessions). In this case, the next data synchronization session will encounter two different versions of the same data. This situation is called "data change conflict".
The data change conflicts are resolved automatically by prioritizing one of the two available data versions; however, you can override the conflict resolution manually. For this, select a list item.

  • Description of the conflict resolution automatically chosen by the application will be displayed at the bottom of the list.
  • To review differences between two data versions, click Show differences.
  • Before you decide which data version should be prioritized, you may want to review both versions. For this, click All actions - Show version created in this infobase or All actions - Show version created in other infobase.
  • To view the list item, click All actions - Change data.
  • Once you decide which data version to keep, click Confirm to preserve the current conflict resolution, or click Revise to change the conflict resolution. This will disable the item and assign the Reviewed attribute to it; the item will be hidden from the list by default.
  • All resolved conflicts are hidden from the conflict list by default. To display these items, click All actions - Show resolved conflicts.
  • To re-review a resolved conflict and make changes to it, click All actions - Remove Reviewed attribute.

Declined due to prohibited date

This list displays data declined because they belong to a date period for which any data import is prohibited.

  • To compare the declined data with the current application data, click Show differences (provided that current data is available in the application).
  • To open an item, click All actions - Change.
  • To review the other data version, click Show rejected version.
  • To import data regardless of any data import prohibitions, click Accept version.
  • To hide the data from the list without making any decision on it, click Ignore. 
  • To display the ignored catalog items, click All actions - Show ignored.
  • To revert the ignored catalog item, click All actions - Do not ignore.
