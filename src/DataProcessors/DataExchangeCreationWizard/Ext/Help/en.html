This wizard is used to set up data synchronization between two applications. You can set up data synchronization respecting exchange rules, ignoring exchange rules, or set up user collaboration in a distributed infobase.

Stages of data synchronization setup

Data synchronization setup including communication channel selection is performed in two stages. Stage 1 of data synchronization setup is performed in this application; then, you need to log on to the other application to continue with the data synchronization setup. To transfer information between the data synchronization setup stages, a temporary file is used. In case of data synchronization using a direct infobase connection or over web service, all setup stages are performed in this application.

The following parameters are defined during the setup procedure:

  • Connection parameters. The parameters provided by user are validated;
  • Data synchronization rules;
  • Default values for data import;
  • Infobase prefix (if not defined earlier).

If you setting up data synchronization respecting exchange rules, the first setup stage will end with data sending test. We recommend that you import data for further object mapping at the end of the second setup stage. For this, select a related check box.

When entering user logon name and password, keep in mind that data synchronization success depends on this user's access rights (for example, if the user has no access to 1C:Enterprise server, data synchronization may fail).

Each step is described in more details below.

Selecting data synchronization setup stage

On this page of the wizard, you choose to start the setup from Stage 1 or Stage 2. Use the radio button to select:

  • Stage 1 - Set up a new data synchronization.
    • Click the Details hyperlink to view the detailed information on both applications participating in this data synchronization.
  • Stage 2 - Resume data synchronization setup. Select this stage to continue with data synchronization setup in the target application.

    • Specify name of the data synchronization settings file. If the settings file was lost, open data synchronization settings from the Data synchronization list, and click All actions - Get data synchronization settings for target application.

  • We recommend that you back up your infobases before performing data synchronization setup. Click the hyperlink to create a backup data copy (recommended).

Click Next to proceed with data synchronization setup.

Selecting connection to target application

Use the radio button to select:

  • Direct connection on this computer or in a local network (in this case, both setup stages are performed in this application).

    • Enter the target application connection parameters:

      • Specify location of the target application:
        • On this computer or in a local network;
        • Select Infobase directory.
      • On 1C:Enterprise server;
      • Enter Server cluster;
      • Enter Infobase name.
  • use radio button to select:
    • 1C:Enterprise authentication:
    • Enter Name and Password of the user on whose behalf data synchronization will be performed.
  • OS authentication.
  • Click Test connection to validate the provided information.
    • Direct connection between the applications over the Internet:
    • Enter the application URL;
    • Enter User name and Password;
    • select the Remember password check box.
    • Click the Internet access parameters hyperlink and configure proxy server parameters, if necessary.
    • Click Test connection to validate the provided information.
      • Other channels (network directories, FTP, email)

          Click Next to proceed to the next data synchronization setup stage.

          Setting up connection parameters using local or network directories

          • Select the Set up connection using local or network directories check box.
          • In the Directory field Select path to a directory located on this computer or in local network.
          • To reduce the size of sent messages, select the Compress and archive sent data check box. Enter the Archive password if necessary.
          • To validate the provided settings, click Test connection.

          Click Next to proceed to the next data synchronization setup stage.

          Setting up parameters for FTP connection

          • Select the Set up connection through FTP check box.
          • Enter Directory path. Port will be filled in automatically.
          • Fill in the User name and Password fields.
          • Enter Maximum file size if the selected FTP resource has limited storage.
          • Select the Passive connection check box if necessary.
          • To reduce the size of sent messages, select the Compress and archive sent data check box. Enter the Archive password if necessary.
          • Click the Internet access parameters hyperlink and configure proxy server parameters, if necessary.
          • To validate the provided settings, click Test connection.

          Click Next to proceed to the next data synchronization setup stage.

          Setting up parameters for email connection

          • Select the Set up connection through email check box.
          • In the Email account field, Select an account from list or click Create (press F8) to create a new account.
          • Enter Maximum email attachment size.
          • To reduce the size of sent messages, select the Compress and archive sent data check box. Enter the Archive password if necessary.
          • Click Test connection.

          Click Next to proceed to the next data synchronization setup stage.

          Setting up data synchronization parameters

          • Enter Description of this infobase;
          • Enter Description and Prefix of target infobase;
          • Select Primary connection method - when multiple connection methods are available, the primary method will have priority.
          • Specify data synchronization rules:
          • The default Data export rules (used when sending data to the other application) are displayed. To change these, click the Change data export rules hyperlink.
          • Specify Data import rules (used to autofill blank fields). This field is mandatory. Click the Change autofill values hyperlink and select values from the list. This field is not available when setting up data sycnhronization that ignores exchange rules.

          Click Next to test the provided parameters and finish data synchronization setup for this application (in case of direct connection between applications, you will need to set up and validate parameters for the other application as well).

          Ready to start data synchronization

          On this page of the wizard, data synchronization settings for the first application are displayed (in case of direct connection between applications, you will need to validate parameters for the other application as well). Validate the settings and click Next to proceed.

          Setup stage 1 completed successfully

          Now, the data synchronization settings will be saved to a file located in a directory selected earlier. In case of direct connection between the application, the data synchronization setup will end now.

          • The file path will be displayed. Click Save.
          • The Send data to target application check box is selected by default.
          • Click Ready to end data synchronization.

          Specifics of data synchronization setup in client/server mode

          When setting up data synchronization in client/server mode infobase, keep in mind that all data will be synchronized on a computer where 1C:Enterprise server was installed.

          For 1C:Enterprise servers under Microsoft Windows, the path looks like this:
          \\servername\resource

          For 1C:Enterprise servers under Linux, the path looks like this:
          //servername/resource.

          Make sure that the user on whose behalf the 1C:Enterprise server runs has sufficient rights to read and write in the data synchronization directory. If the user does not have the sufficient rights, you will need to grant them these rights, or select another user with sufficient rights to read and write in the data synchronization directory and run the 1C:Enterprise server on their behalf. The same recommendations apply to FTP and email access. For more information on 1C:Enterprise server administration, see the 1C:Enterprise platform documentation.

          Specifics of distributed infobase node creation

          During Stage 1, the initial distributed infobase (DIB) image is created. All wizard pages remain as they are. On the Setup stage 1 completed successfully page, make sure the Create initial image of subordinate DIB node check box is selected. However, you can go back to this step at any moment during the data synchronization setup procedure.

          • After selecting the check box, click Done. You will be asked to create the initial DIB image. For this, click the Create initial image hyperlink.
          • By default, the settings file required for DIB node creation will be created in a directory selected for exchange purposes, On this computer or in local network.
          • However, you can select to create it On 1C:Enterprise server via radio button. In this case you will need to enter Infobase parameters:
          • Enter 1C:Enterprise server cluster.
          • Enter Infobase name.
          • Select parameters in the Secure connection field.
          • Select DBMS type.
          • Enter Database name and User password.
          • Specify Date shift.
          • Select Interface language.
          • Select the Set scheduled job lock check box if necessary.
          • Click Create initial image.
          • Save the file containing the initial image of subordinate DIB node to a new directory, which will be further used as a working directory for the new infobase.
          • Create a new infobase via the Add infobase to list command. Specify path to the initial image file. The selected infobase will be added to the infobase list.
          • Once you log on to the application, the wizard will automatically proceed from the Continue setting up data synchronization with master node page. Click Next to proceed with the setup and select a master node connection method.

          Creating DIB nodes (external files)

          If any of your files are stored in volumes, during initial image creation you will need to specify path to the .ZIP archive file containing all files stored in volumes. When starting the application with a local branch infobase for the first time, you will need to execute the Store initial image files command and specify the path to the archive file.
