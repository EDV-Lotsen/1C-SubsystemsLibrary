In this form you can view and change various settings of scheduled job.

It usually is rare when the scheduled job settings should be changed. In most cases Use check box state is changed to enable or disable scheduled job. 

1. Main property group

It contains most often used properties of scheduled job. But as other properties they are set during application design and should be changed only in special cases.

Property name Property purpose
Description Scheduled job description. Usually this property is blank and instead of it the value of synonym is used, that is set in scheduled job template in Designer.
If synonym of template is not set then its name is used.
Use

This check box allows to enable or disable execution of scheduled job.
It is useful when the scheduled job should be disabled for a certain period of time without of changing its schedule.

Repeat if error occured, times Number of attempts of scheduled job execution if an error occurs.
Repeat if error occured after, sec Timeout (in seconds) between attempts to execute the scheduled job if an error occurs.

2. Advanced property group

Property name Property purpose
ID

It can be used only by experienced administrator who knows 1C:Script. It is required to investigate problems in scheduled job execution, if any.

Key

It can be used only by experienced administrator. It is required for fine tuning of scheduled job execution and to investigate problems in scheduled job execution, if any. If Key value is set then before a background job is started, the application checks if there is no other active background job with the same scheduled job template as the current one.

User name Only for client/server mode. The infobase user name which is used for scheduled job execution. For file mode a single user is used for execution of all scheduled jobs, moreover, jobs are executed in privileged mode.

 

3. Metadata property group

Property name Property purpose
Name Name of scheduled job template that is set in Designer.
Synonym Synonym of scheduled job template that is set in Designer.
Method name Name of procedure in configuration that will be executed in background job that will be created for the scheduled job.

 

4. Warnings and error details

This field contains warning texts and error details if warnings and/or errors occured during the last background job execution. The same content is available, if you open the latest background job this scheduled job.
