In this form you can view and change various settings of the scheduled job.

It usually is rare when the scheduled job settings should be changed. In most cases only the Use check box state is changed to enable or disable a scheduled job. 

1. The Main property group

It contains most often used properties of the scheduled job. But as other properties they are set during the application design and should be changed only in special cases.

Property description Property purpose
Description The scheduled job description. Usually this property is blank and instead of it the value of synonym is used, that is set in scheduled job template in Designer mode.
If synonym of template is not set then its name is used.
Use

This check box allows to enable or disable execution of scheduled job.
It is useful when the scheduled job should be disabled for a certain period of time without changing its schedule.

Repeat attempts if error occurs Number of attempts to execute the scheduled job if an error occurs.
Repeat period if error occurs, sec Timeout (in seconds) between attempts to execute the scheduled job if an error occurs.

2. The Advanced property group

Property description Property purpose
ID

It can be used only by experienced administrator who knows 1C:Script. It is required to investigate problems in scheduled job execution, if any.

Key

It can be used only by experienced administrator. It is required for fine tuning of scheduled job execution and to investigate problems in scheduled job execution, if any. If Key value is set then before a background job is started, the application checks if there is no other active background job with the same scheduled job template as the current one.

User name Only for client/server mode. The infobase user name which is used for scheduled job execution. For file mode a single user is used for execution of all scheduled jobs, moreover, jobs are executed in privileged mode.

 

3. The Metadata property group

Property description Property purpose
Name The name of the scheduled job template that is set in Designer mode.
Synonym The synonym of the scheduled job template that is set in Designer mode.
Method name The name of procedure in the configuration that will be executed in a background job created for the scheduled job.

 

4. Warnings and error details

This field contains warning texts and error details if warnings and/or errors occured during the last background job execution. The same content is available, if you open the latest background job of this scheduled job.
