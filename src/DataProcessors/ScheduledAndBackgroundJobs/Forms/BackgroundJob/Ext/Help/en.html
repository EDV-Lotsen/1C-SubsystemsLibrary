Background job properties can be viewed in this form.

1. The Main property group

Property description Property purpose
Description This is a copy of the Description field of the scheduled job and usually is blank. If the Run now command on the command bar of the scheduled job is executed, the description of the scheduled job will have following value: "<Scheduled job presentation> started manually".
State Possible values: Completed, Completed with errors, Canceled by user, Executing.
Start date Date and time when the background job is started.
End date Date and time when the background job is finished. If the background job is still executing the date and time is: <>).
Execute on The name without domain of the computer inside a network where the background job was executed. It either is a server name in client/server mode or a user computer name in file mode.

2. The Advanced property group

Property description Property purpose
ID

It can be used only by experienced administrator who knows 1C:Script. It is required to investigate problems in scheduled job execution, if any.

Key

The value of the Key property that was specified in scheduled job properties (at the moment when the background job was created) or another value that was specified explicitly if the background job was created from 1C:Enterprise script (i.e. not by server scheduler).

Method name The procedure name in the configuration that executed in this background job. It usually is the same value as value of the Method name property of the corresponding scheduled job.

 

3. The Scheduled job property group

Property description Property purpose
Description

Contains one of following values (in order of searching for first filled property):
1. Description.
2. Synonym.
3. Name.

ID The ID of scheduled job that corresponds to the background job.

 

4. Warnings and error details

This field contains warning texts and error details if warnings and/or errors occured during the background job execution.
