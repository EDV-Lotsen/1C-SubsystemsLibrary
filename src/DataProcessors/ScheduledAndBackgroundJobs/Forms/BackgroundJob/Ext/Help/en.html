Background job properties can be viewed in this form.

1. Main property group

Property name Property purpose
Description This is a copy of Description field of scheduled job and usually is blank. If Execute now command on command bar of scheduled job is executed, the description of scheduled job will have a value: "Start manually: <Scheduled job presentation>".
State Possible values: Completed, Completed with errors, Canceled by user, Executing.
Start date Date and time when a background job is started.
End date Date and time when a background job if finished (if job is still executing then date and time is: <>).
Execute on Name of a computer inside a network without domain on which the background job was executed. It either is a server name in client/server mode or a user computer name in file mode.

2. Advanced property group

Property name Property purpose
ID

It can be used only by experienced administrator who knows 1C:Script. It is required to investigate problems in scheduled job execution, if any.

Key

The value of Key property that was specified in scheduled job properties (at the moment when the background job was created) or another value that was specified if background job was created programmatically, but not a server scheduler.

Method name Name of procedure in the configuration that is or was executed in this background. It usually is the same value as value of Method name property of the corresponding scheduled job.

 

3. Scheduled job property group

Property name Property purpose
Description

Value of one of properties (in order of searching for first filled property):
1. Description.
2. Synonym.
3. Name.

ID ID of scheduled job that corresponds to the background job.

 

4. Warnings and error details

This field contains warning texts and error details if warnings and/or errors occured during the background job execution.
