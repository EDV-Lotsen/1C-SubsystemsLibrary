In order to administrate scheduled and background jobs the system provides following functionality:

  1. Scheduled jobs execution monitor;
  2. Scheduled jobs;
  3. Background jobs.

 

1. Scheduled job execution monitor

Control can be simple or detailed.

Simple control - includes analysis of State and End date columns on Scheduled jobs page; it also includes analysis of Messages to user and error details fields on scheduled job edit form.

Detailed control - includes analysis of background jobs on Background jobs tab, that background jobs were created and executed to execute scheduled job.

Detailed control is made by filtering of the list of background jobs by a specific scheduled job and analysing background job attributes. Note, that scheduled job execution details are obtained by selection of a part of details of the last background job execution.

Note: the filter to background jobs is applied only by Refresh command, so after changing filter settings one should execute this command.

2. Scheduled jobs

Command Details
Refresh Refreshes the the list of scheduled and background jobs.
Add Opens a form for creating a new scheduled job.
Change Opens a form for editing a scheduled job.

Copy

 Opens a form for creating a new scheduled job filled with values from the copied scheduled job.

Delete

 Deletes a selected scheduled job in list, except for predefined scheduled jobs.
Schedule Opens a form for editing a schedule of a scheduled job.
Execute now

Executes a procedure of a scheduled job in the current session (for file mode) or in a background job on server (for client/server mode). When a procedure is executed in a background job on server, the last execution date, that is used for execution by a schedule, for scheduled job is not changed.

Scheduled job execution setup

For file mode only. Opens a form with scheduled job execution settings.

Open separate session for scheduled job execution

For file mode only. Opens a separate session for execution of scheduled jobs. The session is opened with options that allow it to process background jobs and when the current session is closed, this separate session will be closed automatically too.

 

3. Background jobs

Filter Details
By state

Allows to filter by a state:
1) Completed,
2) Completed with errors,
3) Canceled by user,
4) Executing.

By period

Allows to filter by a period:
1) Custom - allows to specify a custom period,
2) Without filter - no filter by period will be applied,
3) Night - standard period of 12 hours starting from 9:00 PM till 09:00 AM,
4) Yesterday - standard period of 24 hours starting from  00:00 AM till 11:59 PM of yesterday,
5) Today - standard period of 24 hours starting from 00:00 AM till 11:59 PM of today.

By scheduled job Allows to filter by a scheduled job:
1) The check box enables or disables this filter,
2) Scheduled job name, <not defined> is used to filter background jobs, started for execution of other (not scheduled) jobs.
Command Details
Refresh Updates the list of background jobs with applied filter and a scheduled jobs list.
Open Opens a form with background job details.
Cancel Cancels execution of a selected background job (if it is executed currently).
