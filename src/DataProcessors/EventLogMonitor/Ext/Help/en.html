For administrative purposes it is often required to figure out what events happened in a certain period of time or what actions performed by a user. Event log is intended for this purpose. This log can be set up in Designer.

Event log is not a dynamic list. To see events that occurred after the form was opened you must execute Refresh current list command. Event log displays a fixed number of events that can be set in Event count limit. An arbitrary number can be entered in that field in addition to predefined values 200, 500, 1000, and 10000. Value in this field is saved for future use.

To filter events in the event log you can use Set filter command. Number of shown items will be limited by Event count limit value. To disable the filter use Clear filter command.

You can open the list of active users using Active user list command.

If you need view the saved log (a .lgf file) to open a file use Load from file command in All actions menu. A new window will be opened for viewing.

To change a set of columns in event log table you can use Change form command in All actions menu.

If some infobase object is associated with an event then this object can be opened for viewing. To do that you must move a cursor to Data or Data presentation column of the event and execute Open data for viewing command.
