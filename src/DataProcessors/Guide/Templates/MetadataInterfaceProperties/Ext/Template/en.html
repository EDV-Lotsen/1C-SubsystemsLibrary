The Metadata interface properties

Catalogs, which suppose to have a small number of items, in metadata have the QuickChoice property set. This provides a choice from the drop-down list in input fields, if this mode is not disabled manually for the attribute or the form field.

For the AdditionalInformation attribute of the Counterparties catalog in metadata the MultiLine property value is set. Using this information the catalog item form automatically displays the input field for this attribute as the multi-line field.

On the VendorSettlementAccount of the MoneyExpense document example the link between attributes in the configuration is demonstrated. In the ChoiceParametersLinks property it is set, that when chosing the Settlement Account, is is required to filter settlement accounts by the Owner field, using the Vendor field for filtering.

The Product attribute of the Products tabular section of the ProductReceipt document demonstrates how to set a specific choice parameter value. The "Filter.Kind" choice parameter is set to filter the catalog items of the "Product" kind. Thus, services are excluded from the choice.

For the Product attribute of the tabular section of the ProductExpense document the ChoiceParametersLinks has a BalanceByWarehouse property set linked to the Warehouse attribute. This Parameter is used in the ListFormWithBalance of the Products catalog to display Products for the specific Warehouse. For this link the the LinkedValueChangeMode property is set to DontChange, due to the Product attribute is not required to be changed when the Warehouse is changed.

For the Characteristics information register the Value resource's metadata have LinkByType property set, defining that the value should be of types from the list, which are set in the characteristic, selected in the CharacteristicType dimension.
