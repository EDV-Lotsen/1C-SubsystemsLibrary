Fill checking

For many attributes the FillChecking property is set in the configuration. Basing on this property value on the data input the data is checked and if the data is not filled, errors are displayed to the user.

For example, in the ProductExpense document attributes and the tablular section have the FillChecking property set (to the ShowError value). This allows the system to check automatically the attributes data and the tabular section filling. The checking is performed by the form extension on before the document posting.
In the document module, in the FillCheckProcessing handler, the standard behavior for the Customer attribute is overrided.

Besides, the fill checking overriding is demonstrated in the object module of the Counterparties catalog.
