Dynamic list

In the ListFormWithBalance form of the Products catalog the possibility to work with the complicated query in the dynamic list and capabilities of the conditional appearance are demonstrated. In the list the only products having balances in warehouses are displayed. If the balance is less than zero then the row is displayed in red. Services are highlighted with blue. This Form is used for choosing items in the ProductExpense document. It is specified as the choice form for the Product field. In the form module the received parameter (Warehouse) is set as a query parameter. The parameter transfer is performed due to the ChoiceParameterLinks metadata property in the Product attribute's is set.

In the list form of the ProductReceipt document the ability to group data in the dynamic list is demonstrated. Documents are grouped by the Vendor attribute.

In the list form of the FinancialDocuments documents journal shows how a user can set quick list filter.

In the CurrentCurrencyRates information register form, displayed on the desktop, the ability of the dynamic list to work with the virtual table is demonstrated.

For the ProductReceipt and the ProductExpense documents list forms several filter items are set with disabled usage in properties of dynamic lists. This allows the user to apply a filter immediately during the list setup without of adding it from the fields list.
