Messages and product states

In the ProductExpense document module during posting the products, availability at the warehouse is checked. If there is a shortage of products, a message is generated. This message has information, which allows the system to show on the document's form the place, where to correct the input data. The field name is specified in the message for this purpose, this field should be activated and then the SetData method is called, which adds to the message the information about the path to the object data in the form.

In the DocumentPosting data processor as an example the state displaying and user interrupt checking for long operations is demonstrated.
