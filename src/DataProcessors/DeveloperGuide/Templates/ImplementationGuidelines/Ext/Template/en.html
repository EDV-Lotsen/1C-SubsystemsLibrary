Implementation Guidelines

Additional Reports and Data Processors

You can create reports and data processors, and then add them to the application. For each report or data processor, you have to write the following:

  • AdditionalDataProcessorInfo function
  • AdditionalReportsAndDataProcessorsOverrided module

The module includes the following functions:

  • GetCommonCommandsOfAdditionalDataProcessors – defines the available commands for starting a data processor.
  • GetCommonCommandsOfAdditionalReports – defines the available commands for starting a report.

Exceptions for the List of References

You can define the list of references that are not taken into account during reference-dependent operations (such as object deletion) in the CommonUseOverrided module.

The module includes the following function:

  • GetRefSearchExceptions – returns the list of metadata object names that can refer to different metadata objects without affecting the business logic of the application.
