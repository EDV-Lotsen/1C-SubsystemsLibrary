During development, it may be necessary to update the auxiliary data from time to time.
The list of possible reasons is provided in the form.

1. Generally, the auxiliary data is updated during execution of update handlers, mainly, the required update handlers (version *).

To update the auxiliary data as fast as possible, you can limit the procedure to execute the required handlers only (version *). Certain application properties will be monitored for changes. If no changes are found for some of the properties, the corresponding update phases will be skipped to optimize the execution time.

If you want to execute the required update handlers (version *) only:
a) specify the StartInfobaseUpdate start parameter in the command line (using /C option) or in Designer;
b) set the Start infobase update constant and restart the application
    (not supported in SaaS)

2. In certain situations (especially during active debugging), handler execution can encounter errors.
In this case, some of the data will be updated while some will not. To ensure complete auxiliary data update
you need to execute the update through this processor, though this may take longer.
