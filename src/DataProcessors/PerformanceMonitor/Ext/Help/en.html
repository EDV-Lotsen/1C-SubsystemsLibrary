This form is used to evaluate application performance.

The application features standard tools for performance data collection and analysis in accordance with Apdex (Application Performance Index, international standard for software performance evaluation).

Before you can proceed with performance evaluation, you need to fill in the Key operations list.

Software performance evaluation based on Apdex methodology includes the following major stages:

  • Defining priorities for each key operation supported by the application;
  • Defining target execution time for each key operation;
  • Gathering information of execution time for each key operation;
  • Calculating the performance score based on the gathered data, and interpreting the results.

To view or analyze the performance evaluation results, execute the Performance indicators command located in the Administration - Support and maintenance section.

Table

The performance indicators are displayed in a table with the following columns:

  • Key operation - key operation name. The table lists all key operations supported by the application. Key operation names are copied from the Key operations list. Plus, a predefined key operation Overall system performance is displayed in the table but not in the list by default.
    • Use buttons  Add and  Delete to prepare a list of key operations to be used for performance evaluation.
    • After a selected key operation was added to the table, it will become unavailable for further selection, thus preventing the users from adding it to the table again. 
  • Priority - copied from the Key operations list.
    • Use buttons  Move up and  Move down to rearrange the key operations list.
    • When moving a key operation up or down, its Priority will be adjusted automatically.
    • The topmost operation in the list has the highest priority, while the bottommost operation has the lowest priority. To increase priority of a key operation, move it up the list; to decrease priority, move it down.
    • The Overall system performance record is always located at the top of the list. It has no priority; it is used to estimate overall system performance.
  • Target time - copied from the Key operations list. Performance estimate directly depends on this value. Chaning target time values will affect the resulting performance score. The target time is measured in seconds. 

Period settings

  • Before performance evaluation, you need to specify the evaluation period.
    • Enter Start date and End date for the evaluation period. Click  to select a date from calendar; set time if necessary (the default time is 0:00:00).
  • The Step field defines how many results are displayed per period. Example: if you set Period to 1 day and Step to 1 hour, 24 results per key operation will be displayed. The hourly results will be calculated in accordance with Apdex methodology. If a key operation was not performed during  a 1-hour step, the corresponding table cell will display zero. The measurements are performed during execution of each key operation (for example, during document posting).
  • The measurement results are displayed in the Table and Chart tabs of the Performance monitor window

Updating table contents

The list is not updated in real time. To refresh the table or chart contents after making adjustments to the Period or Step field values, execute the Update performance indicators command manually.

Calculating target time

Select a key operation (with the exception of Overall system performance), and execute the Calculate Apdex target time command available via context menu accessed by right-clicking. Target time cannot be calculated until time measurements are performed for the key operation. 

Performance monitor

Filtering performance indicators

To set a performance indicator filter, click  Set filter and use a radio button to select:

  • Filter - Above or Below;
  • select Apdex score: Good, Fair, or Poor;
  • This will automatically set Condition to the numeric equivalent of the selected Apdex score.
  • Click OK to enable the filter.

Key operation execution history

To view Key operation execution history, double-click a corresponding value in the table. This will display measurement period, key operation parameters copied from the key operation list, and the number of measurements. The measurement table contains Duration, End time, and User for each measurement.

Chart

To view the table data in visual form, go to the Chart tab, where data related to each key operation is displayed as a separate diagram.

Exporting performance measurement data

  • The measured performance data can be exported to external files in background. This can be useful, for example, when you need to run performance analysis for an external application.
  • To set up automated export of the performance data, click All actions - Settings. The generated external files, as well as files with application settings used for performance testing, are compressed to ZIP archive automatically.
  • To perform manual export of the performance data, click All actions - Data export.
