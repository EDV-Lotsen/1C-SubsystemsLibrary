This form is used for automated text extraction management.

If one or more working processes run on a Linux server, and one or more client computers are connected via web client, the infobase can contain files that did not undergo text extraction procedure and therefore cannot be found by using full-text content search. In this case, you need to start thin or thick client on any client computer running under Microsoft Windows, and execute the Full-text search management - Start text extraction command located in the Administration - Support and maintenance - Scheduled operations section. Alternately, you can execute the Automatic text extraction command located on actions panel of the Organizer section.

Parameters setup

  • Set Execution time interval (60 seconds by default);
  • Specify Number of files per batch (100 by default);
  • Number of files with unextracted text is calculated automatically.

Text extraction

You can submit files for text extraction:

  •  batch by batch.
    • Click Start to start text extraction from file batches (can be aborted at any time);
    • Click Stop to abort the procedure.
  • all files at once.
    • Click Extract all.
