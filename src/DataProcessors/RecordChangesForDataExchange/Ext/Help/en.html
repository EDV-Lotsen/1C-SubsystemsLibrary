This list is used to record application data changes. All data changes must be recorded before the data is sent to other application for synchronization purposes.

To access the list, use the All actions - Sent data command in the Data synchronization list.

All message numbers are recorded. To view or edit a message, click a hyperlink displaying its number.

You can apply filters to the data change list. Select from list:

  • All;
  • Only exported;
  • Only not exported.

The list consists of two sections:

  • The left section includes two columns:
    • Configuration object - this column lists applied objects (dictionaries, documents, etc) as a tree;
    • Recorded/not exported - this column displays the number of items in these objects that were changed (or recently created) but not yet exported.
  • Content of the right section varies depending on the object currently selected in the left section:
    • If multiple objects are selected, the Details column will contain detailed statistics for the selected objects.
    • If a single object is selected, the right section will be divided into two columns:
      • Object description (for example, Currency) - this column displays all list items that were changed but not yet exported;
      • Sent No - number of the sent message containing the changed list items (this column displays a message number or Has not exported message).

Recording changes, canceling recording

All changed and created objects are recorded automatically. To record an object manually, select the object in the tree by using its check box, and click Record changes: 

  • - all objects of selected types - you can select the object itself and all objects subordinate to it;
  • - objects of selected types with auto-recording;
  • - single object (create the object, or select an object from list);
  • - filtered objects (set a filter and select the required items);
  • - query results.

To cancel sending all data prepared for other applications, select the object tree root (Infobase) and click Cancel change recording - All objects of selected types. In this case, the settings are identical to the above.

Other available actions:

  • To set up data change recording parameters, click Setup. This allows you to set up auto-recording parameters, assign message number for recording purposes, or specify path to the query console directory.
  • To view the export results, select the object with recorded changes in the left section. In the right section, a list of object items will be displayed. Select a required item and click Show export results, or use the All actions - Standard export results command.
