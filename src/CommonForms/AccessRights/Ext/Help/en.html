This list is used to view and edit contents of the user (external user) access groups.

To open this list, execute the Access rights command located on the user (external user) navigation panel.

User's personal access rights are defined by roles assigned to this user via access groups and profiles.

An access group defines a set of actions with the application data the members of this group are allowed to perform. Generally, access groups correspond to user job functions or duties. A user can be a member of multiple access groups at the same time, these groups collectively defining their personal access rights settings.

Access group setup can be performed by:

  • administrators - for all application users;
  • user (external user) list managers - for all users included in the corresponding lists, except the access groups with administrative rights.
  • access group member list managers - only for users included in the corresponding access groups.

The list displays the following information:

  • Name - access group name;
  • Profile - access group profile name;
  • Responsible - name of the responsible manager.

The list includes the following tabs:

  • Access groups - commands used to manage access rights settings for a specific user, and a list of access groups containing this user.
  • Allowed actions (roles) - all actions allowed to the user (external user). The list of roles assigned to a user includes all roles assigned to access groups containing this user.
    • The role list can be sorted in alphabetic order or grouped by subsystem. To switch between these views, click By subsystem.

Access rights settings

  • Include in group - add user to the member list of an access group.
    • Select an access group from list and click Select, or double-click the access group.
  • Exclude from group - remove user from the member list of selected access group.
  • Change group - modify settings for a selected access group.

Access rights report

To view the full list of user access rights, including the full list of applied objects and access rights per object granted to this user (user group), User access rights report can be generated. Execute the Access rights report command.
