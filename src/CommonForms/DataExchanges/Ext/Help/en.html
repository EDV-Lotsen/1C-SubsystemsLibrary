This form is used to set up data synchronization and store the data synchronization settings.

The Data synchronization window can be accessed from the application desktop or from the Data synchronization section, by executing the related command. If no data synchronizations—or only a single data synchronization—was created, the window will be displayed as a panel. In this case, the All actions menu will be hidden and all necessary commands and fields will be displayed directly on the panel. If multiple data synchronizations were created, the window will be displayed as a list.

The window contains the following information:

  • - data synchronization is performed on schedule. Data synchronizations automatically performed under a premade scenario are also marked with this icon in this column;
  • Application - description of the application participating in this data synchronization;
  • Infobase - description of the infobase participating in this data synchronization;

  • Synchronization status (minutes elapsed since the last synchronization attempt, Now, Never) is also displayed here. The failed synchronization attempts are marked with icon:

    • Data sent - time the latest data was sent;

    • Data received - time the latest data was received.

Setting up data synchronization

To set up a new data synchronization with an external application, execute the related command from the Set up data synchronization submenu. Data synchronization setup window will be displayed.

Viewing setup information

For more information on the accounting specifics in applications participating in data synchronization, and for recommendations on data synchronization setup, click All actions - More on data synchronization with this application.

Data synchronization

Select a setting and click Synchronize, or double-click a setting, to start the automated data synchronization procedure. Click Yes to confirm selection.

Manual data synchronization

For manual data synchronization, select a setting from the list and click Synchronize with additional parameters... This will import new data from the other application and then export the new data to it. Moreover, you can use the Synchronize with additional parameters... command to perform identical data mapping prior to synchronization. This method is useful when identical data was independently entered into both applications for any reason. To reduce this risk, you can set up frequent data synchronization on schedule.

Data synchronization on schedule

To Set up data synchronization on schedule, execute the All actions - Data synchronization scenarios command. In this case, data synchronization in client/server mode will run in the background, in the form of Data synchronization background job. However, when in file mode, you will need to start a separate session to perform background jobs.

Viewing warnings

Follow the Warnings hyperlink (followed by number of warnings) for detailed information on the warnings that occurred during data synchronization.

Show only failed attempts

To view only the failed data synchronization attempts, click Only failed.

Viewing event log

Detailed information on data synchronization execution is stored in the application event log. To view the list of data synchronization events, or events themselves, execute the All actions - Send data events or All actions - Receive data events command.

Using external rule files instead of standard object conversion and object change record rules

To use external rule files instead of standard object conversion rules and object change record rules, you will need to:

  • Open data synchronization for which you want to use the external rules.
  • Execute the All actions - Import object conversion rules or All actions - Import object change record rules command, according to your needs.
  • In the object conversion or object change record window, set the radio button to Rules from file and click Import rules to locate a rule file stored on the local computer or in a network directory.
  • Make sure that rules contained in the selected file are compatible with current versions of source and target applications.
  • Click Save and close. The imported rules will apply immediately.

Deleting data synchronization settings

Select data synchronization settings and click All actions - Delete synchronization settings. Attention! This will delete the settings irrevocably.

Restoring infobases from backup

When restoring infobases from backup, we recommend that you:

  • Temporarily disable the automated data synchronization by clearing the Use schedule check box for each data synchronization scenario;
  • Cancel sending data to other applications for synchronization purposes (for each data synchronization settings, you will need to execute the All actions - Sent data content command, then select the object tree root in the Recording changes windows, and execute the Cancel change recording - All objects of selected types command);
  • Re-import data that was entered into other applications since the creation of this infobase backup copy;
  • Enable the automated data synchronization.

If a significant data volume was lost, we recommend that you delete all data synchronization settings and set up the data synchronization once more. Otherwise, other applications participating in data synchronization may receive the outdated data from the restored infobase, resulting in data loss.
