To set up a new data exchange with another node or with another application select the appropriate command from submenu Create. New data exchange setup wizard will be opened.

To execute data exchange manually select one of settings in list and execute Execute now command. The new data from another application will be imported first and exported back to that application after that. Besides using Setup - Execute with settings command you can execute data exchange with mapping of identical data. This is useful in cases when in both nodes (applications) the same data was inputed independently somehow. To lower probability of such situations you can set up a data exchange by a frequent schedule.

Using Setup - Automatically by schedule the data exchange by a specific schedule can be set up. In this case in client/server mode the data exchange will be executed in background using Data exchange execution scheduled job. In file mode a separate session for scheduled job execution should be started.

Replacing standard object conversion rules and object change record rules with rules from external file

To replace standard object conversion rules and object change record rules with rules from external file you must:

  • Open the data exchange for which rules are required to be changed.
  • Select Open object conversion rules or Open object change record rules from Data exchange parameters submenu in the command bar of data exchange form depending on what rules are required to be changed.
  • In rules form select Rules from file option and select a file with rules on local computer or in network directory after clicking Import button.
  • Make sure that imported rules are correct and suitable for current versions of source and target applications. 
  • Click Write and close button. The imported rules will be applied right away.
