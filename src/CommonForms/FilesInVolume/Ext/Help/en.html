This list is used to view files stored in disk volumes.

Files in each volume are stored in subdirectories. For each file version creation date, a new subdirectory is created. All stored files retain their original names and extensions, with version number added in the middle. Example: original file name Orders2.txt, version number 2, resulting file name Orders2.2.txt.

To access the list, execute the Files in volume command located in the volume navigation panel.

The list includes:

  • Path to file - full file name including the name of subdirectory where the file is stored;
  • Size - file volume, in bytes;
  • Author - name of user who added the file.

Filtering files

  • You can filter files by the related applied object. For this, select a value from list in the Files owner field. Example: select Attached files (Meeting) as owner to display all files attached to Meeting documents.
  • To store a file in the application, it is not necessary to have it linked to an object. Information on these files is available in the file list. To view files in this list, select File versions as owner.

Viewing file information

To view file information, double-click the file. This will open the attached file card, or file version card, depending on the owner (as specified in the Files owner field).
