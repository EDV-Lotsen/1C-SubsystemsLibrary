This list is used to manage files attached to applied objects.

To open it, execute the Attached files command located on navigation panel of the applied object.

The list includes:

  •  - signed with DS.
  •  - encrypted.
  • Name - full name of the file. An icon next to the name specifies file type.
  • Details - detailed file information.
  • Change date - date of the latest file data change (local time);
  • Author - user who added the file;
  • Size (KB) - file size;

Additional data can be displayed via the All actions - Customize form command:

  • Extension - file format; also specified by an icon.
  • Edits - user editing the file; and more. 

Adding new files

  • You can Add new files stored on the hard disk.
  • After selecting a file on the hard disk, the Attached file window displaying the file properties opens automatically.
  • The attached files are stored in dedicated directories. A separate directory is created for each applied object.
  • In some lists, items that have attached files are marked with  icon.

Viewing file cards

  • Click  (press F2) to open the Attached file window and view properties of the selected file in list; alternately, you can execute the All actions - File properties command.

Saving files

  • Click  to save an attached file to a specified directory on a local or network disk.
  • Alternately, you can execute the All actions - Save as command.
  • To open the file directory in Windows Explorer, execute the All actions - Open file directory in Windows Explorer command.

Updating files

  • To update an attached file from the original file stored on hard disk (the source of the attached file), execute the All actions - Update from file on hard disk command.

Editing attached files

  • In thin client, the .mxl spreadsheet document files are edited in an embedded form of the spreadsheet document editor. In web client, you need to download an application to edit the spreadsheet documents. For more information on editing spreadsheet documents in web client, follow this hyperlink.
  • To edit files in other formats, you will need to have corresponding applications installed; for example, to edit files in Microsoft Word format, you will need to install Microsoft Word, etc.
  • The Open command opens an attached file in an external application in editing mode (if the file was not locked) or in viewing mode (if the file is locked for editing by another user).
  • The Edit command opens the attached file in editing mode and locks the file. The locked files are unavailable to other users for editing; they are marked in green in the attached files list.
  • The End edit command unlocks the attached file, saves it, and clears the "locked for editing" mark. When finished editing, save the file and close the external editing application.
  • The All actions - Cancel edit command unlocks the attached file without saving the changes, and clears the "locked for editing" mark. This is useful if you do not want to save the changes you made (for example, your changes contain errors).

DS and encryption

The DS and encryption dropdown menu contains a set of commands for digital signature management:

  • Sign - create a digital signature for the file. Before creating the digital signature, you will be offered to select a DS certificate from the list of available certificates. Alternately, you can select the certificate during application customization. The digitally signed files cannot be edited; they are marked with icon.

  • Add DS from file - add a digital signature to a file. This could be useful, for example, when the digital signature file was obtained after the signed file.

  • Save with DS - save digital signature to a separate file on hard disk. This could be useful, for example, when you need to deliver a signed file to another user.

  • Encrypt - encrypt a file. Unlike file signing procedure, multiple certificates can be used for file encryption, for example, when you need to encrypt a file for several users. The encrypted files are marked with icon.

  • Decrypt - decrypt an encrypted file. To decrypt a file, you need to type in the private key access password.
