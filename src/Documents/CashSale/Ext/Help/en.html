The cash sale document incorporates a sales invoice and a cash receipt into a single transaction. It is essentially an ad hoc sales document.

To enter a cash sale

  1. In the Sales section, click Cash sales and then click Create.
  2. Enter or select the following:
    • Number. By default the system assigns consecutive numbers when documents are saved, and no user input is required.

      You have the option to enter a custom number instead. Document numbers can include prefixes. For example, if you specify the number "CS-1", your next cash sales are automatically numbered "CS-2", "CS-3", and so on. 
    • Currency. The transaction currency.

      This field is only available if the Multicurrency check box is selected in Accounting settings.
    • Bank account. The account (e.g. bank, cash, credit card, PayPal, etc.) used for the transaction.
    • Price includes VAT. Indicates whether the prices specified in the cash sale include VAT.

      This field is only available if the VAT check box is selected in Accounting settings.
    • Date. The cash sale date.

      This field is prefilled with the current date.
    • Exchange rate (shown as currency symbols, for example, $/€). The exchange rate between the transaction currency and the reporting currency.

      The system retrieves the exchange rate from the Exchange rates list.

      If you change the cash sale date, the system recalculates the exchange rate. If you change the date or currency, the system does not recalculate prices in the line items, therefore it is recommended that you specify the date and currency before entering the line items.

      This field is only available if the Multicurrency check box is selected in Accounting settings.  
    • Ship to. The shipping address.

      This field is prefilled with the customer's default shipping address.
    • Deposit type. Select one of the following:
      • Undeposited funds. The payment is deposited later individually or grouped with other payments using the Deposit document.
      • Account. The payment is deposited by this document.
    • Customer. The customer (here you can only select companies of the customer type).

      The field name can be customized (for example, to Client) in the Customer name field in Accounting settings. 
    • Company code. The company code (read-only).

      This field is only available if the Display company codes check box is selected in Accounting settings.
    • Location. The location of the warehouse where the sold items are stored.

      This field is only available if the Multilocation check box is selected in Accounting settings. 
    • Payment method. The payment method.
    • Ref #. Optional transaction number (for example, payment ID/number).
  3. In the Line items list, select the items included in the cash sale, and specify their prices and quantities.

    Item prices are automatically prefilled from the Price list.

    If the VAT check box is selected in Accounting settings, a default VAT code is prefilled for each item.

    If the Sales tax (US) check box is selected in Accounting settings, the default sales tax is prefilled for each item (by default, inventory items are taxable and noninventory items are nontaxable).

    The system automatically calculates a line total for each item. If the Sales tax (US) check box is selected in Accounting settings, it also automatically calculates the sales tax based on the buyer's tax code selected in the Company window.

  4. In the Memo field, you can enter notes on the cash sale.
  5. Click Post and close.

    The informational (read-only) section at the bottom of the form contains the following data:
    • Sales tax. The sales tax based on the sales tax code assigned to the customer company and the sales tax types assigned to line items.
    • VAT. VAT calculated based on the VAT codes assigned to items.

      When you post the cash sale, a VAT liability is recorded to the General journal.

      This field is only available if the VAT check box is selected in Accounting settings.
    • Total FCY. The total amount in the transaction currency.

      This field is only available if the Multicurrency check box is selected in Accounting settings. 
    • Total. The total amount in the reporting currency.
