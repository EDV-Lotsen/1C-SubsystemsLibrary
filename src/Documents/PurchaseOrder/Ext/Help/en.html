The purchase order is the central document in the purchasing workflow. The system tracks received and invoiced items against purchase orders.

Purchase orders do not generate General journal transactions.

To enter a purchase order

  1. In the Purchasing section, click Purchase orders and then click Create.
  2. Enter or select the following:
    • Number. By default the system assigns consecutive numbers when documents are saved, and no user input is required.

      You have the option to enter a custom number instead. Document numbers can include prefixes. For example, if you specify the number "PO-1", your next purchase orders are automatically numbered "PO-2", "PO-3", and so on. 
    • Currency. The transaction currency.

      This field is only available if the Multicurrency check box is selected in Accounting settings.
    • Order status. The purchase order status (read-only). The following statuses are available:
      • New. The purchase order is being created.
      • Open. The items have not been received or invoiced.
      • Backordered. The items are partially received and invoiced.
      • Closed. The items are received and invoiced.
    • Date. The purchase order date.

      This field is prefilled with the current date.
    • Exchange rate (shown as currency symbols, for example, $/€). The exchange rate between the transaction currency and the reporting currency.

      The system retrieves the exchange rate from the Exchange rates list.

      If you change the purchase order date, the system recalculates the exchange rate. If you change the date or currency, the system does not recalculate prices in the line items, therefore it is recommended that you specify the date and currency before entering the line items.

      This field is only available if the Multicurrency check box is selected in Accounting settings.  
    • Vendor. The vendor (here you can only select companies of the vendor type).

      The field name can be customized (for example, to Supplier) in the Vendor name field in Accounting settings. 
    • Company code. The company code (read-only).

      This field is only available if the Display company codes check box is selected in Accounting settings.
    • Location. The location of the warehouse where the purchase is delivered.

      This field is only available if the Multilocation check box is selected in Accounting settings. 
  3. On the Line items tab, select the items included in the purchase order, and specify their prices and quantities.

    Item prices are prefilled with item last costs. The system automatically calculates a line total for each item.

    If the VAT check box is selected in Accounting settings, a default VAT code is prefilled for each item. 

    The Backorder field is filled automatically with the quantity that is not yet received or invoiced. If the order has the Open status, the Backorder quantity is 0. The Received and Invoiced columns are filled automatically when a purchase invoice based on the purchase order is created and posted.
  4. Enter or select the following:
    • Memo. In this field you can enter notes on the purchase order.
    • Price includes VAT. Indicates whether the prices specified in the purchase order include VAT.

      This field is only available if the VAT check box is selected in Accounting settings.
  5. Click Post and close.
     

The informational (read-only) section at the bottom of the form contains the following data:

  • VAT. VAT calculated based on the VAT codes assigned to items.

    When you post the purchase invoice based on this purchase order, a VAT liability is recorded to the General journal.

    This field is only available if the VAT check box is selected in Accounting settings.
  • Total FCY. The total amount in the transaction currency.

    This field is only available if the Multicurrency check box is selected in Accounting settings. 
  • Total. The total amount in the reporting currency.
