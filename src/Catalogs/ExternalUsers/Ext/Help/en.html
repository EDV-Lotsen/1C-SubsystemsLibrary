This list contains accounts of users that work with this application from the outside. These users are usually very limited in access rights, they do not actively input data, work separately and do not interact between each other.

The main method of creation and editing of external users is by using External access command that is placed in command bar of some objects, for example in the item form of counterparty or individual. An external user is created and assigned to the current object or a form of existing external is opened during this command execution.

In case when a large number of external users this list can be grouped, for example by types of relations with them: partners, counterparties and so on. To view all external users including ones that are placed in subordinate groups, you must set Show external users of subordinate groups on the bottom of the list. To add a new group of external users use commands on the left side of the list. To add a large number of external users to a group you can use Fill command. One external user can belong to several groups of external users.

For administration of external user list the Administration right is required. The responsible for administration of external users can use External access command, edit External users and External user groups lists. All users can view information about their users and change some properties, such as: password, application language and contact information.

External users can be marked as inactive. To view those external users you can be use Show inactive external users command in All actions menu of the list of external users.
