In the catalog External users there are users that have the entrance to IB from outside. Usually their access is strongly limited, they don’t take part in account processes, work apart, don’t interact with each other.

Catalog is designed to maintain a list of external users. Typically the elements of catalog are connected to relevant informational base users. With the "Information base users" command system administrator can perform an analysis of conformity of such catalog elements as External users and Information base users.

Basic way of adding and editing catalog elements with command External access, that is located in the command bar forms of objects informational base, for example, contractors, individuals. When running command immediately the catalog element is creating and connecting to informational base or the existing element is opening.

External users can be classified, for example on relations with them: partners, contractors and so on. To do this you have to create required quantity of external user groups and add external users to the groups in the left part of the list.

External user without administrative rights has access only to information about himself. He also has rights to change some features, such as: name, password, language and contact information.

Editor of an external user may use command External access, edit catalogs External users and External users groups, but may not edit access rights directly (and only with changing the group content of external users).
