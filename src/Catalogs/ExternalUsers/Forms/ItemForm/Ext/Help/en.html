To edit external users you should have administration or responsible for external users right. If you do not have these rights, you are allowed only to view your user data and change your password, application language, and contact information.

An external user is linked to some object in the application, for example a counterparty or an employee. This object is specified in Object field.

Invalid check box is cleared by default for all external users. But if a user is not not allowed to use an application any more and at the same time this user can not be deleted because of existing references to this user object, it is recommended to set Invalid check box. This user will not be available to be selected as a task performer or responsible for document, it will not be displayed in external user lists.

To allow an external user to log on the application you must set Infobase access allowed check box and set name, password and other authorization options for that user.

Recommendations for user passwords:

Strong password must:
- is at leas eight characters long,
- contains uppercase latin letters,
- contains lowercase latin letters,
- contains numbers,
- contains symbols (underline, brackets and so on).

A strong password should not look like bulk of characters, not like something readable.

Good password examples:
"nj7{jhjibq*Gfhjkm,
F5"GhkmNjNjn;t{HI.

It is not recommended to use passwords that match with user name, contain only numbers, words, or repeating character sets. 

Bad password examples:
Smith,
qwerty,
12345678,
123123123.
