Description of file storage volume properties.

When creating a new storage volume, access to directory specified in the Full path field is validated automatically. If the specified path is invalid, a warning message will be displayed.  Path to an existing volume cannot be modified.

These fields are mandatory for each volume:

  • Name - full name of file storage volume;
  • Code - filled in automatically;
  • Full path
  • For 1C:Enterprise servers under Microsoft Windows - specify path to the volume directory on Microsoft Windows server, in UNC format (\\Computer name\Directory name\File name\);

    or

      • For 1C:Enterprise servers under Linux - specify path to the volume directory on Linux server;
    • Fill order - enter an integer. The default value is 1. Storage volumes are filled in this order (ascending) until each volume is full. You can exploit this feature in specific cases. For example, you can disable a volume temporarily by increasing its fill order number.
    • Max size - here you can specify maximum disk size allowed for file storage purposes on this volume (volume quote). When total size of all files stored in a volume exceeds the quota set for this volume, the next volume in fill order will be used for file storage. If none of the available volumes have sufficient free capacity for file storage, this event will be added to the event log and an error message will be displayed. To disable the total file size limitation, set volume quota to 0. 
    • Current size, Mb - total size of files currently stored in the volume. Updated automatically as the volume fills up;
    • Comment - enter detailed volume information in this field.

    Specifics of filling in the Full path field under Linux

    • For 1C:Enterprise servers under Linux, you can use the local path to the file storage volume instead of the UNC path. Example: /home/path. Before you can use UNC path, you need to mount the directory.  Log in as administrator and execute the following commands:
      • mkdir directory, where directory is your directory's name. Example: go to the /home directory and execute mkdir shared. This will create the shared directory (full path: /home/shared).
      • mount -t cifs //server/share directory -o //domain/user, password=pwd, where
           - //server/share - UNC path to another computer, directory - path to directory on Linux-based computer (example: \home\shared), //domain/user - domain/user name if required, pwd - user password if required.
      • If the user name and password are not required, simply execute mount -t cifs //server/share directory.
    • Now you can specify directory (example: /home/shared) as the path to the volume.

    Viewing volume file lists

    • Files in each volume are stored in subdirectories. For each file version creation date, a new subdirectory is created. All stored files retain their original names and extensions, with version number added in the middle. Example: original file name Orders.txt, version number 2, resulting file name Orders.2.txt.
    • To view the list of files in volume, execute the Files in volume command located in the volume navigation panel.
