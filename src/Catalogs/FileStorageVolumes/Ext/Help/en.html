This list is used for file storage volume management.

It is only utilized when the In volumes on hard disk file storage mode is active. To access this list, select the Store files in volumes on hard disk check box located in the Administration - File operation settings section. This will enable the File storage volumes command located in the Administration - File operation settings and Organizer sections. Execute this command to open the list.

The list includes:

  • Name - the name of file storage volume;
  • Fill order - an integer. Storage volumes are filled in order (ascending) until each volume is full;
  • Max size - here you can specify maximum disk size allowed for file storage purposes on this volume (volume quote). To disable the total file size limitation, set volume quota to 0. When total size of all files stored in a volume exceeds the non-zero quota set for this volume, the next volume in fill order will be used for file storage. If none of the available volumes have sufficient free capacity for file storage, this event will be added to the event log and an error message will be displayed; 
  • Full path for Microsoft Windows - path to the volume directory on Microsoft Windows server, in UNC format (\\Computer name\Directory name\File name\);
  • Full path for Linux - path to the volume on Linux server;
  • Comment - detailed information on the volume.

Creating file storage volumes

  • Click  Create and fill in the required fields.

You must specify one or several directories located on hard disks of server running the application or other servers in the network, to be used for file storage.

Managing file storage volumes

  • When creating a new storage volume, volume path specified in the Full path field is validated automatically. If the path is invalid, a warning message will be displayed. 
  • Storage volumes are filled in order (ascending) until each volume is full. You can exploit this feature in specific cases. For example, you can disable a volume temporarily by increasing its fill order number.
  • Path to an existing volume cannot be modified.
  • A volume cannot be deleted while it is being referenced by any file versions.
  • Volume access errors are recorded to the event log and can be viewed by the application adiministrator.
  • Files in each volume are stored in subdirectories organized by date. A new subdirectory is created for every day. All stored files retain their original names and extensions, with version number added in the middle. Example: original file name Orders.txt, version number 1, resulting file name Orders.1.txt.
  • Toggling file storage mode does not affect the currently stored files; it only defines how the new files will be stored. For example, if you enable the In volumes on hard disk file storage mode, this does not move the currently stored files to volumes; only new files will be stored in volumes. To move the currently stored files from infobase to volumes, execute the Transfer files to volumes command located on actions panel of the Organizer section.
  • If data synchronization is enabled between two applications and the source application stores files in volumes on hard disk, the initial image creation procedure must include creation of a compressed file containing the file storage volumes.
  • Accordingly, the procedure of initial image import to the target application must include specifying location of the compressed file with the file storage volumes. To extract the volume-based files from the compressed file, execute the Store initial image files command located on actions panel of the Organizer section.

Comment
To start using volumes located on network disks, you need to run the 1C:Enterprise 8 server on behalf on domain user (such as domain1\user1) with access rights to volume location specified as UNC path. This user must be a member of the Users group on the local computer, have Run as service and Run as batch job rights, and have access rights to directory where system files of 1C:Enterprise 8 server (event log, temporary files, etc) are stored.
