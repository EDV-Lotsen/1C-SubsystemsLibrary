This application allows to keep the following contact information kinds:

  • Address - an attribute with this contact information kind is intended to store an address as an arbitrary string or as an address template. Allows to fill an address using Address classifier in address input form. If the address is filled using Address classifier then ZIP/Postal code is filled automatically from Address classifier after the address input is finished. Predefined actual and legal addresses are used in print forms of documents.
  • Phone - an attribute with this contact information kind is intended to store phone numbers as an arbitrary string or as phone number input template. The predefined phone is used in print forms of documents.
  • Email - an attribute with this contact information kind is intended to store email addresses. This attribute is used during sending emails from the application.
  • URL - an attribute with this contact information kind is intended to store web page URLs.
  • Fax - an attribute with this contact information kind is intended to store fax numbers as an arbitrary string or as fax number input template. This attribute values are used in print forms of documents.
  • Other - an attribute with this contact information kind is intended to store an arbitrary text of contact information, for example information how to find the office of the company.

If a contact information attribute has Address kind it becomes possible to set Edit in dialog only property. This property allows to limit number of input methods of new and edit methods of existing addresses: an address input form or selection from Address classifier can be used in that case. If Only home country address property is set the home country can be selected for address.
