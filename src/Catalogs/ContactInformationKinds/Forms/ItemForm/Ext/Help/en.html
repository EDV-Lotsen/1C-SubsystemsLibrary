Description of contact information field properties.

The application is distributed with a predefined list of contact information fields for specific documents and lists. For these predefined contact information fields, only the settings are available for editing; all other attributes are filled in automatically by default and cannot be edited.

Contact information types

This application allows to keep the following contact information types:

  • Address - an attribute with this contact information kind is intended to store an address as an arbitrary string or as an address template. Allows to fill an address using Address classifier in address input form. If the address is filled using Address classifier then ZIP/Postal code is filled automatically from Address classifier after the address input is finished. Predefined actual and legal addresses are used in print forms of documents.
  • Phone - an attribute with this contact information kind is intended to store phone numbers as an arbitrary string or as phone number input template. The predefined phone is used in print forms of documents.
  • Email - an attribute with this contact information kind is intended to store email addresses. This attribute is used during sending emails from the application.
  • URL - an attribute with this contact information kind is intended to store web page URLs.
  • Fax - an attribute with this contact information kind is intended to store fax numbers as an arbitrary string or as fax number input template. This attribute values are used in print forms of documents.
  • Other - an attribute with this contact information kind is intended to store an arbitrary text of contact information, for example information how to find the office of the company. This field can contain up to 1024 text characters.

Adding contact information fields to lists or documents

Fill in the required information:

  • Select сontact information Type from list.
  • Enter Name to represent this contact information field in the list or document.
  • Enter Tooltip text to be displayed when hovering mouse pointer over the contact information field.
  • Select Group if not selected by default. Each group corresponds to a specific application document or list where you want to add a new contact information field.

Setting up contact information fields

  • Select the Mandatory check box to ensure that the contact information field is not left blank when entering documents or lists. If this check box is selected, users will be prohibited from saving their data until the contact information field is filled.
  • Select the Allow multiple value input to enable multiple value input option. For example, you may need to add multiple email addresses, phone numbers, etc, for a specific organization or individual.
  • The Edit in dialog box only check box is available for the following contact information types: Address, Phone, Fax. Select this check box to force the application users to create or edit addresses, phone or fax numbers by template only (address classifier will still be available for address selection). Clear this check box to allow free-format address input. This is reserved for emergencies when you may need to knowingly enter an invalid address (for example, an address on an incoming document is invalid and cannot be used, but you still have to enter it). Such address will be stored as a free-format address, and its usage will be limited accordingly. Later, you will be able to make corrections to the invalid address by editing it by template.

Notes on the Address field setup 

  •  Select the Home country only check box to prohibit selecting any foreign countries during address input. We recommend that you select this check box if none of your employees have foreign addresses; this will expedite the address input procedure. Selecting this check box makes additional check boxes available:
    • Prohibit invalid address input - if this check box is selected, you will not be able to save the address data until the address is validated. We recommend that you only use this option when input templates and address classifier are implemented. To enable this option, select the Edit in dialog box only check box.
    • Hide obsolete addresses - select this check box if you want to view the lists without the addresses considered obsolete on the latest address classifier load date. These addresses are identified with icon in the lists . The check box is cleared by default.
    • Include country in presentation - select this check box if you want the country name to be displayed when full address presentation is generated automatically.

Notes on the Email address field setup

For fields of Email address type, the entered email address data can be validated automatically. Select the Prohibit incorrect address input check box to ensure that only valid email addresses are entered.
