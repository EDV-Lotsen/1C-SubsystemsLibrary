When a user (or external user) is written with Access to infobase allowed check box set, the corresponding new infobase user is automatically created or the existing one is modified. And on the contrary, when a user (or an external user) is written with Access to infobase allowed check box cleared the corresponding infobase user is deleted automatically from the infobase user list. However there is a number of situations when the infobase user list becomes out of synch. In that case this list can be used for the following tasks:

  • Find invalid infobase users and synchronize user and infobase user lists between each other:
    • if the infobase user was added using Designer (or programmatically bypassing the recommended mechanism), it will be suggested to delete this infobase user;
    • if the infobase user was deleted using Designer (or programmatically bypassing the recommended mechanism) and there is row in users list for that user, it will be suggested to open and overwrite the user info to remove reference to the deleted infobase user;
    • if full name of the existing infobase user was modified using Designer (or programmatically bypassing the recommended mechanism), it will be suggested to open and overwrite the user info to restore the full name of the infobase user.
  • View the list of users and external users that do not have corresponding infobase users (the ones who have Access to infoinfobase allowed check box cleared).
  • View the list of all infobase users (the same as in Designer).

 

To open this list you can use Infobase users command in All actions menu of users (or external users) list.
