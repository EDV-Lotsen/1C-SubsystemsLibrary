To edit users you should have administration right. If you do not have this right, you are allowed only to view your user data and change your password, application language, and contact information.

Full name of a user is displayed in all lists and reports. Invalid check box is cleared by default for all users. But if a user is not not allowed to use an application any more and at the same time this user can not be deleted because of existing references to this user object, it is recommended to set Invalid check box. This user will not be available to be selected as a task performer or responsible for document, it will not be displayed in user lists.

To allow a user to log on the application you must set Infobase access allowed check box and set name, password and other authorization options for that user. Name for logging on the application can differ from full name for that user, for example: Doe as a name and John Doe as a full name.

Recommendations for user passwords:

Strong password must:
- is at leas eight characters long,
- contains uppercase latin letters,
- contains lowercase latin letters,
- contains numbers,
- contains symbols (underline, brackets and so on).

A strong password should not look like bulk of characters, not like something readable.

Good password examples:
"nj7{jhjibq*Gfhjkm,
F5"GhkmNjNjn;t{HI.

It is not recommended to use passwords that match with user name, contain only numbers, words, or repeating character sets. 

Bad password examples:
Smith,
qwerty,
12345678,
123123123.
