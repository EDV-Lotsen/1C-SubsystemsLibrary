User Groups

Catalog is used to configure user groups.

Folders are created as a flat or a hierarchical list. Users of the nested groups enter into the owner-folders indirectly in the hierarchical list.

Folders are generated according to the enterprise structure, and also according to positions and project groups, for example:

  • Enterprise
    • Administration
      • Department 1
      • Department 2
    • Administration Heads
    • Technical Departments
      • Department 1
      • Department 2
        • Group 1
        • Group 2
      • Technical Department Heads
    • Production
      • Shop 1
        • Area 1
        • Area 2
      • Shop 2
        • Area 1
        • Area 2
        • Area 3
      • Production Heads
    • Enterprise Heads
    • General Inventory Group

There is a predefined group All Users, which includes all users indirectly.

Destination of user groups is a possibility to specify both a user and a user group in some applied goals.
Because user groups are supposed to be stored taking into account users administrative combining, they can be used, for example:

  • In access rights and restrictions configuration.
  • In business-processes in the performer roles.
  • In other areas of some applied solution.

It is recommended to define maintenance of the user group list in the enterprise documentation.
The administrator modifies user list and user group content.
Such approach would allow to use user groups by different infobase users in intelligent manner.
