Generally, access group profiles are set up during applied solution deployment and configuration phase. The profiles are predefined sets of allowed activities (roles), described in terms of object access rights and restrictions of these rights (access kinds) as corresponding to relevant access groups.

However, in certain cases, predefined profiles prove to be insuffucient for the purpose of comprehensive setup of user access rights. In these cases, the administrator is free to modify the predefined access group profiles, or add new profiles as necessary.

Profiles are used to:

  • create a common role list applying to a group of application users. For example, the Manager access group profile, containing the following roles, can be applied to all managers:
    • Create and edit customer orders,
    • View reference documentation.
    • ...
  • specify access kinds to govern access to specific catalog items, documents, etc. accessible under the above roles. For example, the Manager profile can be set up to allow, or prohibit, access to customer orders, reference documents, and other application data related to specific companies, partners, business operations, and other available access kinds. Settings for specific companies, partners, business operation types, and other access kinds available in the profile are managed via corresponding access groups.

Access kinds and access restrictions

Not every role supports access restrictions by access kind. For example, the Create and edit customer orders role allows to restrict access only to customer orders related to specific companies or partners; while the Start web client role always allows to start web client and does not support any additional access conditions. When managing access kinds in a profile, please keep in mind that the profile must contain a comprehensive set of access kinds sufficient for all roles linked to the profile. If an access kind is not linked to the profile, it cannot be used to define any access restriction conditions.

Application administration profile

A predefined profile named Administrator is available in the application, containing a single role named Full rights. This role grants full unrestricted access to the application data. We do not recommend that you add the Full rights role to any other profiles, as this will automatically override any access restrictions and transform the profile into a functional copy of the Administrator profile.

The Administrator profile is linked to the predefined access group named Administrators. We do not recommend that you create any other access groups based on this profile.
