Description of access group profile properties.

A variety of predefined access group profiles is available in the application. However, in certain cases, predefined profiles prove to be insuffucient for the purpose of comprehensive setup of user access rights. In these cases, the administrator is free to modify the predefined access group profiles, or add new profiles as necessary.

Generally, an access group profile contains several roles and describes access kinds that can be used by the administrator to set up user access rights via access groups. In other words, an access group profile is a template used by the administrator for access group management purposes.

One or several access groups can be based on a single profile. For example, two access groups: Headquarters accountants and New York branch accountants, can be created on the basis on the Accountant profile, the former group granting access to Headquarters and the latter to New York branch.

Creating access group profiles

  • Enter profile Description.
  • To add an access group profile to a folder, fill in the Group (folder) field. Access group profiles can be grouped by a variety of criteria. Click  Select and select a folder from list.
  • Select Allowed actions (roles) on the corresponding tab.
  • Specify Access restrictions on the corresponding tab.
  • Enter Comment (any additional information on the access group profile), if necessary.

Allowed actions (roles) 

When creating a new access group profile, you first need to assign a list of roles allowed for this profile, according to the demands of the target users.

A role defines a set of actions with the applied objects the user is allowed to perform. Some roles may correspond to user job functions or duties, some - to more specific functions. For example, the Accountant role grants access to a set of functions needed to perform accountant's duties, while the Read files role allows viewing file contents. Roles are often used to allow viewing or editing specific data stored in the application.

  • Go to the Allowed actions (roles) tab and select check boxes for the required roles.
  • Click  to select all roles in the list.
  • Click  to clear all selected roles.
  • Click Selected roles only to show selected roles only.
  • Click By subsystem to display the role list:
    • sorted alphabetically;
    • grouped by subsystem.

Access restrictions

The Access restrictions tab is used to assign a list of access kinds to the profile.

An access kind is a rule governing access to application data. Access kinds are used to precisely define the data area allowed to the access group members.

In an elementary case, no access kinds are assigned to a profile and data access rights are solely governed by profile roles. For example, if a profile includes the Product receipt document view role, all members of access groups based on this profile are allowed to view these documents. However, once you add the Companies access kind to this profile, you will have the option to allow the users to view documents for a specific company only, to block the users from viewing documents for a specific company, etc. To reiterate, the roles assigned to a profile define which data or functionality is accessible and which is not; while the access kinds assigned to a profile allow to define more precise conditions for data accessibility. We recommend that you assign access kinds to a profile whenever you need to specify complicated access rights for the group members.

  • Specify Access kinds. Click Add. Click  to select an access kind from list. To delete a record, click .
  • For each access kind assigned to a profile, you can select any of the four available Access values:
  • All prohibited, exceptions defined in access groups - select this access value to have access to all data prohibited by default. To allow access to an object, an exception must be created in an access group.
  • All allowed, exceptions defined in access groups - select this access value to have access to all data allowed by default. To prohibit access to an object, an exception must be created in an access group.
  • All prohibited, exceptions defined in profiles - select this access value to have all data access prohibited by default. Data access exceptions can be created in profiles only; access rights cannot be modified on access group level. These access kinds will be hidden from the access groups.
    • When this access value is selected, exception creation functionality will be enabled. Click Add. Click  to select values from relevant lists.
  • All allowed, exceptions defined in profiles - select this access value to have all data access allowed by default. Data access prohibition exceptions can be created in profiles only; access rights cannot be modified on access group level.
