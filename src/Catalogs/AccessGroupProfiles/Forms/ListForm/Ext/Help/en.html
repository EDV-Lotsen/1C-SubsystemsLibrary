This form is used for access group profile management.

To facilitate management of multiple access group profiles with similar purpose and avoid confusion, you can consolidate similar access group profiles into folders.

To access the list, execute the Access group profiles command in the Administration - User and rights settings section.

All access group profiles are broken down into two categories:

  • Primary profiles define a set of access rights sufficient to perform specific activities in the application.
  • Additional profiles are used to grant users additional rights not covered by the primary profiles.

When a primary profile (Accountant, Warehouse manager, etc) does not provide sufficient rights they should be complemented with an additional profile (Counterparty communication, Unposted document printing, etc).

Recommendations on creating primary access group profiles

A primary profile should include a set of roles that grants all access rights required by users during their routine duties and operations, but does not provide them with excessive or unadvisable access to the application data or functionality. Moreover, a primary profile should include additional roles not directly related to the user duties but yet necessary to perform these duties. For example, the following roles must be included in the profile intended for document entry operators, besides the primary role allowing to create and edit these documents:

  • a role allowing logon via thin client, thick client, or web client;
  • predefined role Basic rights;
  • roles allowing to view catalogs containing items selectable in the document fields;
  • roles allowing to view document processing reports, etc.

For example, the Auditor profile should include the Read individuals primary role to allow viewing personal data on individuals, and a set of additional roles to support basic work functionality (such as Basic rights, Start thin client, etc).
