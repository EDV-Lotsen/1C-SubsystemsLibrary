This list is intended for accounting external user groups.

External user groups can be used for grouping large number of external users and when the group must be specified, for example, when setting up access rights.

Groups can be flattened or hierarchical. In a hierarchical list, external users of nested groups are implicitly included in parent groups.

Groups are created by application functional like in the following example:

  • Counterparties
    • Partners
    • Main customers
    • Main vendors
    • Poll members
  • Individuals

The All external users predefined group implicitly includesall external users.

You can create groups of external users of a single type. These groups can implicitly include all users of a single type.

When you defining roles directly, you can define a role for a whole external user group.
