Description of access group properties

An access group defines a set of actions with the application data the members of this group are allowed to perform. Generally, access groups correspond to user job functions or duties. A user can be a member of multiple access groups at the same time, these groups collectively defining their personal access rights settings.

Access groups can be used to create and modify common access rights settings for a group of users performing similar activities in the application, for example, Accountants.

Administrator rights are required for access management. The user assigned as responsible manager of an access group is allowed to modify member list of their access group on the Members tab.

An access group is always linked to an access group profile commonly incorporating multiple roles. Once a user is added to an access group, they are automatically assigned all roles listed in the access group profiles. For example, a predefined access group Administrators is linked to the Administrator profile incorporating the Full rights role. This role grants unlimited access to all data; it is assigned to all application administrators belonging to the Administrators access group.

Creating access groups

To create an access group, you need to select a profile (we recommend that you start from this point), specify group member list, and define any access restrictions.

  • Select Profile. The profile selection determines available rights (according to its role list) and available rights restrictions broken down by access kind: for example, restrictions related to Companies, Warehouses, etc. Any access kinds found in the profile will be copied to the access group automatically.
  • Enter access group Name that should reflect its purpose and restrictions by access kind (for more information, see access group description).
  • We recommend that you choose a plural name containing the profile name (as displayed in the Profile field) and clearly describing the access right settings assigned to this access group. For example, you can create access groups named Sales managers in Forward Thinking Inc. and Sales managers in New Tech LLC on the basis of the Sales managers profile.
  • You can add a profile to an access group profile folder. In the Group (folder) field, click  to select a folder from list.
  • Click  to select Member type. According to the access group purpose, Member type should be set to Arbitrary (this allows to add any users or external users to the access group) or otherwise.
    • To limit your selection to a specific list, set Member type to Common users, External users (Demo: Counterparties), External users (Individuals), or External users (Demo: Partners).
    • You can also add external users and external user groups to the access group (this may be useful for survey purposes, etc).
    • Set member type to Arbitrary if you need to mix both common users and external users in the same group.
  • Go to the Members tab and fill in the access group member list.
  • Go to the Access restrictions tab and enter additional access rights settings.
  • You can also enter detailed information on the access group into the Description field.

      Attendees

      • According to the selected Member type, corresponding user lists will be available on this tab for access group member selection.
        • To add multiple users, click Pick; to add a single user, click Add. Select the required access group members from list.
        • To delete one or several access group members, select them and click  Delete.
      • Fill in the Responsible field by selecting a user from the Users list. The user assigned as Responsible is allowed to change member list of their access group.

      Access restrictions

      On the Access restrictions tab, you need to enter additional access rights settings. Access restrictions can only be created on profile level. On access group level, access restrictions can only be elaborated, if allowed by the profile settings. In certain cases, profile-based access restrictions are complete and require no elaboration. In these cases, the Access restrictions tab will be disabled.

      Content of the access kind list on this tab depends on the profile specified in the Profile field. Moreover, profile determines behavior of access kind settings: read-only data access, or allow editing.  For example, if an access group profile includes the Product receipt document view role, all members of access groups based on this profile are allowed to view these documents. However, if the profile includes roles that allow adding and modifying documents, user with this profile will also be allowed to edit these documents.

      The Access kind field is used to precisely define the data area accessible to the access group members. An access kind is a rule governing access to application data. For example, you can use the Companies access kind to allow  the user to view documents for a specific company only, to block the user from viewing documents for a specific company, etc.

      To set up the access restrictions, you need to specify a value list for each access kind.

      • To fill in the values, click  , or double-click. Specify Access values for each Access kind.
      • Two values are available: 
        • All prohibited. In this case, access to all data is prohibited by default; you will need to explicitly specify any allowed values. We recommend that you use this method when you need to grant the users access to a specific limited data area.
        • All allowed. In this case, access to all data is allowed by default; you will need to explicitly specify any prohibited values. We recommend that you use this method when you need to grant the users access to all data except specific areas. In this case, users automaticaly have access to any new data entered into the application. For example, if you set up the Companies access kind to prohibit the users from accessing documents belonging to Forward Thinking Inc., and then you create a new company (New Tech LLC) in the application, the users will not be prohibited from accessing any documents belonging to New Tech LLC. Please keep in mind that "allow" settings have priority over "prohibit" settings. If users from the above example are allowed to access documents belonging to Forward Thinking Inc. via any other group based on the same profile, they will have access to all documents belonging to this company.
      • You can specify a list of values or value groups to serve as exceptions for any Access kind, if necessary. For this, click Add in the Allowed (Prohibited) values list. Select exceptions from lists.

        • In certain cases, it is expedient to specify item groups (instead of single items) in the prohibited/allowed value list. This facilitates the access rights setup procedure. For example, if you set up the Partners access kind to prohibit access to some of the partners, you can specify partner access groups instead of single partners. For this, you will need to create the list of partner access groups, and assign each partner to a group. These activities are not limited to administrators; any user with sufficient rights to edit the partner list can perform this.

        • For some access groups, the Users (or External users) access kind can be used for access rights setup. This only applies to the documents (and other applied objects) containing fields that can be used to specify users allowed to access these documents (Responsible, Author, Performer, etc). To grant group members access to all restricted data available to a specific user, you need to add this user to the allowed value list for the Users access kind. For example, if access to Product receipt documents is governed by the Responsible field value, document posting will be allowed both for users specified in this field, and users included in the allowed list for the Users access kind. The "prohibit" setting follows the same logic.
