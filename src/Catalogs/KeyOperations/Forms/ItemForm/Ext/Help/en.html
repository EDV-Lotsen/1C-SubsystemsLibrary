Description of key operation parameters.

A number of predefined key operations are available in the application. More can be added if necessary. Key operations may include any common operations supported by the application, such as document posting, report generation, and more. To evaluate application performance, you need to manually fill in parameters for each key operation (as these parameters are not filled in automatically after installation).

Key operation parameters

  • Name - this field is filled in for predefined key operations; for each user-created key operation, you need to enter its name manually. This field is mandatory;
  • Priority - specify priority (an integer) for each key operation. No key operations can share the same priority. This field is mandatory;
  • Target time - specify time (in seconds). This field is mandatory;
  • Minimum level - select a value from list (for example, Fair).

Application performance tests are based on these key operation parameters.

Viewing time measurements

Application performance tests include time measurements performed for each key operation. To view the history of time measurements for each key operation, click Time measurements on the navigation panel.
