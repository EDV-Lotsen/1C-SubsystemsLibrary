This list is used to set up sets of additional properties.

You can set up additional property sets for specific applied objects. 

Placement of each additional property in the applied object window and its storage method depend on the property type:

  • Custom fields - stored directly in the catalog item or document (sometimes in a dedicated section or on a dedicated tab). The custom fields are created to implement properties that are considered to be an integral part of the object. They are filled in when the object is edited, and can be edited by any user that can access the object with properties.
  • Custom data - used to store data on catalog items or documents that is not considered to be an integral part of the object. Generally, can be viewed and edited even by users not allowed to access the object itself. The custom data is edited in a separate window accessible from the object window.

Custom fields and data can be used in different combinations for different applied objects: custom fields only, custom data only, or both.

To start using additional properties, select the Common custom fields (data) check box in the Administration - Common settings section. This will give you access to the related commands.

All operations with custom fields (data) are performed in the lists accessible via Custom fields or Custom data commands in Administration - Common settings or Integrated subsystems (part 2) sections. 

The list of custom field sets can also be accessed from applied objects with custom fields set up, via the All actions (or More, in Taxi interface) - Change custom data content command.

The list of custom data sets can also be accessed via the All actions (or More, in Taxi interface) - Change custom data content command from the Custom data window of applied object.

The Custom fields (data) list is a table consisting of two sections:

  • The left section contains Custom field (data) sets assigned to catalogs or documents displayed as groups. If a catalog includes groups, Custom field (data) sets will correspond to the catalog groups.
  • The right section contains a list of custom fields (data) included in the selected set, displayed as a table. Mandatory attributes (data) in the list of custom fields (data) will be displayed in bold. The table contains: 
    • Custom field (data) - contains list of descriptions of custom fields (data);
    • Value type - specifies type and number of values for custom fields (data).

Predefined custom field (data) sets correspond to the applied objects (catalogs or documentss) where custom fields and data can be used. New sets cannot be added in this list. Sets are created automatically when you create groups in corresponding catalogs.

You can create custom field (data) sets both for all catalog items and for specific catalog items. In this case, the catalog will be considered a group, and catalog attributes or catalog attribute values will be used as group items (the number of group items will be equal to the number of attribute values).

Assigning new custom fields (data) to catalogs or documents

Depending on the settings, one or the other command may be available for new attribute (data) creation:

  • Select a set that corresponds to this catalog or document, and click  Create. By default, a custom field (data) can only be added to a single set. Fill in the required fields.
  • Select a set that corresponds to this catalog or document, and click Add - New (identical to the Create command). Using this command, an custom field (data) can only be added to a single set. Fill in the required fields.

Common sets of custom fields (data)

Predefined Common sets are available for specific catalogs. By default, custom fields and data added to such sets will become common for all items of these catalogs.

  • Select a Common set.
  • Click  Create or Add - New. Fill in the required fields.

Adding values by sample

To gain access to this functionality, select the Attributes and data with common value list check box in the Administration - Common settings section.

You can reuse existing attributes and data from other sets, or add attributes with a common value list for multiple attributes of the same type (example: Door color attribute based on Color attribute by sample). In this case, value list of the sample attribute will be used. Preparing value lists for multiple attributes of the same type with a single action saves time and efforts.

  • Select a set from the list.
  • Click  Add - By sample.
  • Select a custom field (data) from the available list. Type and all values of the custom field (data) set as sample will be copied to the new item. Fill in the required fields. By default, the selected attribute will become common with the sample set. Entering new values will not be allowed in this case. You will be notified that this value list is common with another attribute.

Adding common custom fields (data)

  • To gain access to this functionality, select the Common custom fields (data) check box in the Administration - Common settings section.
  • An attribute is considered "common" when it is included in multiple sets. This option is used when it is necessary to share both an attribute and its value list. In lists and reports of catalog/document items, the common attributes are grouped together and displayed as a single aggregated attribute in a separate column or field. Example: the Color common attribute can be used to display all clothes and shoes items of yellow color.
  • Select a set from the list.
  • Click Add - Common.
  • Select a common custom field (data) from the list. If the required attribute is not available in the list, click Create. Fill in the required fields.
  • To add a common attribute to multiple sets, repeat these actions for each set.

Notes on deleting common custom fields (data)

  • To delete a common custom field (data), you need to delete it from each set that includes it. It may be also necessary to clear values of the custom field before it can be deleted.

  • After a property is marked for deletion from a set, the property values will be stored in objects until cleared. The property marked for deletion from a set will be distinctly displayed in the object window or custom data window.

  • Deletion marks can be removed if necessary.
