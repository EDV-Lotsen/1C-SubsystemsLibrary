Performer roles can be used in business processes for role-based task routing purposes. In this case, tasks are initially assigned to roles (example: Room supervisor) rather than to specific users. Associations between users and roles are defined in the Roles and performers list, on which basis the tasks are automatically reassigned to specific users. However, in certain cases association between role and users is not precise and performers cannot be determined. In these cases, you need to explicitly specify the primary and additional addressing objects.

Roles with no performers assigned are color-coded in red.

The External role attribute indicates that list of performers for this role is maintained in another infobase. Roles with the External role attribute set are used for infobase integration purposes.

The Task control manager predefined role can be used to exchange overdue or unassigned task notifications, for all or selected addressing objects. By default, users with the Task control manager predefined role receive notifications on tasks assigned to any performers. If you need to track overdue or unassigned tasks for specific addressing objects only, you need to have the Task control manager role assigned to you for these addressing objects. Example:

* If you revoke the Task control manager role from S. Smith and R. Roberts, J. Johnson will start receiving notifications on their addressing objects as well.

Notifications to users with the Task control manager role are sent to the email addresses specified in contact information provided by these users.
